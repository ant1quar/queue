# Задача:
Необходимо спроектировать класс, реализующий очередь. 

## Вводные
* Среда исполнения: NodeJS 18
* Нативный JS с поддержкой ESM
* Для запуска используем npm start 
* Допускается использование встроенного отладчика

## Требования

* Очередь должна принимать сообщения и к каждому применять определенный параметром конструктора обработчик
* Обработчик просто должен подождать 1 секунду и вывести в stdout пришедшее сообщение
* **Для упрощения:** тип сообщения не принципиален
* У очереди должно быть имя
* Одновременно очередь может брать несколько задач вв обработку, количество должно задаваться параметром конструктора
* В очередь необходимо будет передать 30 произвольно сформированных сообщений и получить соотвествующий вывод в терминале
* При добавлении задачи в очередь или завершении задачи, состояние очереди должно быть записано в файл {queue_name}.json, где {queue_name} - имя очереди
* Также состояние очереди должно быть записано в файл при получении сигналов SIGINT или SIGTERM
* Если файл {queue_name}.json уже существует на старте приложения, то очеоредь должна быть при инциализации восстановлена из него

## Бонусы
* Конкуррентное выполнение задач
* Корректная обработка ошибок
